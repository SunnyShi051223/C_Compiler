# Micro-C 编译器 — 词法分析、语法分析与代码生成 Pipeline 说明

## 一、Pipeline 总览

```
源程序 (.txt)
    │
    ▼
┌─────────────────────┐
│   词法分析 (Lexer)    │  字符流 → Token 序列
│   手工状态机          │
└─────────┬───────────┘
          │ tokens.txt (二元式)
          ▼
┌─────────────────────┐
│  语法分析 (Parser)    │  Token 序列 → 语法分析树
│  简单优先文法 (PSE)   │
└─────────┬───────────┘
          │ 语法树 + 归约动作
          ▼
┌─────────────────────┐
│  语义分析 + 代码生成  │  语法树 → 四元式 → Jack VM
│  语法制导翻译         │
└─────────┬───────────┘
          │ quads.txt + target.vm
          ▼
     目标代码输出
```

## 二、阶段一：词法分析

### 2.1 目标

将源程序字符流转换为 **Token 序列**（二元式），跳过空白符和注释，识别词法错误。

### 2.2 方法

采用**手工状态机**（Hand-written Scanner），逐字符扫描，根据当前状态和输入字符决定转移。

### 2.3 Token 类型定义

```cpp
enum class TokenType {
    KEYWORD,        // 关键字
    IDENTIFIER,     // 标识符
    NUMBER,         // 整型常数
    CHAR_LITERAL,   // 字符常数
    OPERATOR,       // 运算符
    DELIMITER,      // 界符
    UNKNOWN,        // 非法字符
    END_OF_FILE     // 文件结束
};

struct Token {
    TokenType type;    // 类别
    std::string value; // 值
    int line;          // 行号
    int column;        // 列号
};
```

### 2.4 词法规则

| 类别 | 规则 | 示例 |
|------|------|------|
| 关键字 | `int char double struct if else while return read write main` | `int` `while` |
| 标识符 | `[a-zA-Z_][a-zA-Z0-9_]*` | `x` `Point` |
| 整型常数 | 连续数字 | `42` `1024` |
| 字符常数 | `'c'` 或 `'\n'` 等转义 | `'A'` `'\n'` |
| 运算符 | `+ - * / > < == = .` | `+` `==` |
| 界符 | `( ) { } ; [ ]` | `{` `;` |
| 注释 | `//` 单行、`/* */` 多行 | 跳过 |

### 2.5 状态机流程

```
开始
 │
 ├─ 空白符(空格/Tab/换行) → 跳过，继续
 ├─ // → 跳过到行尾
 ├─ /* → 跳过到 */
 ├─ 字母/_ → 读取标识符 → 查关键字表 → KEYWORD 或 IDENTIFIER
 ├─ 数字 → 读取连续数字 → NUMBER
 ├─ ' → 读取字符/转义 → CHAR_LITERAL
 ├─ + - * / > < = . → OPERATOR
 ├─ ( ) { } ; [ ] → DELIMITER
 ├─ 其他 → UNKNOWN（报错，跳过）
 └─ EOF → END_OF_FILE
```

### 2.6 容错机制

- 非法字符（如 `$`、`@`）→ 记录错误坐标，跳过继续扫描
- 未闭合字符常数（如 `'A` 缺少 `'`）→ 报错
- 未闭合多行注释（`/*` 无 `*/`）→ 报错
- 标识符以数字开头（如 `123abc`）→ 报错

### 2.7 输出产物

**文件：`output/Lexer/tokens.txt`**

```
Line  Col  Type          Value
--------------------------------
2     1    KEYWORD       int
2     5    IDENTIFIER    x
2     7    DELIMITER     ;
3     1    IDENTIFIER    x
3     3    OPERATOR      =
3     5    NUMBER        0
3     7    DELIMITER     ;
```

每一行是一个**二元式**：`(类型, 值)`，附带行号列号用于错误定位。

## 三、阶段二：语法分析

### 3.1 目标

判断 Token 序列是否符合语法规则，构建**语法分析树**，报告语法错误。

### 3.2 方法：简单优先文法 (Simple Precedence Grammar, PSE)

简单优先文法是一种**自底向上**的移进-归约分析方法。核心思想：

1. 为文法中的每一对符号（终结符/非终结符）定义优先关系：`<`（低于）、`=`（等于）、`>`（高于）
2. 分析时根据栈顶终结符和当前输入的优先关系，决定**移进**还是**归约**

### 3.3 产生式 (Productions)

产生式是语法规则的形式化表示，格式为 `左部 → 右部`。

```
(1)  P → S ; P                 程序 = 语句序列
(2)  P → S ;
(3)  S → int id                变量声明
(4)  S → char id
(5)  S → double id
(6)  S → struct id { P }       结构体定义
(7)  S → id = E                赋值语句
(8)  S → id ( E )              函数调用（参数为表达式）
(9)  S → id ( id )             函数调用（参数为标识符）
(10) S → id ( )                函数调用（无参数）
(11) S → while ( E ) { P }     循环语句
(12) S → if ( E ) { P } else { P }  选择语句
(13) S → return E              返回语句
(14) E → E + E                 加法
(15) E → E - E                 减法
(16) E → E * E                 乘法
(17) E → E / E                 除法
(18) E → E < E                 小于
(19) E → E > E                 大于
(20) E → E == E                等于
(21) E → E . id                成员访问
(22) E → E [ E ]               数组下标
(23) E → ( E )                 括号表达式
(24) E → id                    标识符
(25) E → num                   常数
(26) E → char                  字符常数
```

**符号说明：**

| 符号 | 含义 | 类型 |
|------|------|------|
| P | Program（程序） | 非终结符 |
| S | Statement（语句） | 非终结符 |
| E | Expression（表达式） | 非终结符 |
| id | 标识符 | 终结符 |
| num | 整型常数 | 终结符 |
| char | 字符常数 | 终结符 |
| int, double, struct, if, while, return | 关键字 | 终结符 |
| +, -, *, /, <, >, ==, =, . | 运算符 | 终结符 |
| (, ), {, }, ;, [ ] | 界符 | 终结符 |

### 3.4 简单优先文法的三个条件

| 条件 | 含义 | 本文法 |
|------|------|--------|
| 产生式右部互不相同 | 无两条产生式右部完全一样 | ✅ |
| 无 ε 产生式 | 无空产生式 | ✅ |
| 无两个相邻非终结符 | 右部中不存在两个非终结符连续排列 | ✅ |

### 3.5 分析步骤

#### 步骤一：计算 FIRSTVT 集合

**FIRSTVT(A)** = 从非终结符 A 出发，所能推导出的**第一个终结符**的集合。

计算规则（不动点迭代，直到集合不再变化）：

| 规则 | 条件 | 结论 |
|------|------|------|
| 规则1 | A → a... | a ∈ FIRSTVT(A) |
| 规则2 | A → B a... | a ∈ FIRSTVT(A) |
| 规则3 | A → B... | FIRSTVT(B) ⊆ FIRSTVT(A) |

**计算结果：**

```
FIRSTVT(E) = { (, *, +, -, ., /, <, ==, >, [, char, id, num }
FIRSTVT(P) = { ;, char, double, id, if, int, return, struct, while }
FIRSTVT(S) = { char, double, id, if, int, return, struct, while }
```

**含义示例：** FIRSTVT(E) = `{ (, *, +, ... }` 表示表达式 E 可以以 `(`、`*`、`+` 等开头。

#### 步骤二：计算 LASTVT 集合

**LASTVT(A)** = 从非终结符 A 出发，所能推导出的**最后一个终结符**的集合。

计算规则：

| 规则 | 条件 | 结论 |
|------|------|------|
| 规则1 | A → ...a | a ∈ LASTVT(A) |
| 规则2 | A → ...a B | a ∈ LASTVT(A) |
| 规则3 | A → ...B | LASTVT(B) ⊆ LASTVT(A) |

**计算结果：**

```
LASTVT(E) = { ), *, +, -, /, <, ==, >, ], char, id, num }
LASTVT(P) = { ; }
LASTVT(S) = { ), *, +, -, /, <, =, ==, >, ], char, id, num, return, } }
```

**含义示例：** LASTVT(S) = `{ ;, }, ... }` 表示语句 S 可以以 `;`、`}` 等结尾。

#### 步骤三：构建优先关系表

遍历每个产生式右部的相邻符号对 (Xi, Xi+1)，建立优先关系：

| 条件 | 关系 | 含义 |
|------|------|------|
| Xi ∈ VT, Xi+1 ∈ VT | Xi **=** Xi+1 | 两个终结符相邻，等优先 |
| Xi ∈ VT, Xi+1 ∈ VN | Xi **<** a, ∀a ∈ FIRSTVT(Xi+1) | 终结符低于非终结符的第一个终结符 |
| Xi ∈ VN, Xi+1 ∈ VT | a **>** Xi+1, ∀a ∈ LASTVT(Xi) | 非终结符的最后一个终结符高于后继终结符 |

栈底 `$` 的关系：
- `$ < a`，∀a ∈ FIRSTVT(P) — `$` 低于程序的首符号
- `a > $`，∀a ∈ LASTVT(P) — 程序的末符号高于 `$`

**特殊关系**（标准算法无法处理括号匹配，手动添加）：

```
( = )    [ = ]    { = }
```

**优先关系表片段：**

```
      $   (   )   *   +   -   .   /   ;   <   =   ==  >   [   ]   id  num  char  ...
$         <                       <               <   <       <   <   <
(     <   =   <   <   <   <   <               <       <   <       <
)                 >   >   >   >   >   >   >       >   >   >
*     <   >   >   >   >   <   <   >   <       <   <   <   >   <   <
+     <   >   <   >   <   <   <   >   <       <   <   <   >   <   <
;     >                       <                       <   <       <   <   <
id    =   =   >   >   >   =   >   >   >   =   >   >   =   >   =
```

#### 步骤四：移进-归约分析

**算法流程：**

```
初始化: 栈 = [$]

循环（直到输入耗尽）:
  1. 跳过词法错误 Token（UNKNOWN）
  2. 提前接受检查：栈 = [$ P] 且 输入 = $ → 接受
  3. 找栈顶最上层终结符 b
  4. 查优先关系表 relation(b, a)，其中 a 为当前输入：

     ┌─────────────────────────────────────────────┐
     │ relation(b, a) = "<" 或 "="  →  移进 (SHIFT) │
     │   将 a 压入栈，读取下一个输入                  │
     ├─────────────────────────────────────────────┤
     │ relation(b, a) = ">"          →  归约 (REDUCE)│
     │   从栈顶弹出符号，匹配某个产生式右部，         │
     │   将左部非终结符压入栈                        │
     ├─────────────────────────────────────────────┤
     │ 无关系                        →  报错 (ERROR) │
     │   记录语法错误，跳过当前输入继续               │
     └─────────────────────────────────────────────┘

输入结束，归约剩余:
  若栈 = [$ P] → 接受
  否则 → 报错
```

### 3.6 分析示例

输入：`int x ; x = 1 ;`

```
Step  栈                   输入           关系        动作
────  ──────────────────   ──────────     ────        ────
 1    $                    int x ; ...    $ < int     移进 int
 2    $ int                x ; ...        int > id    归约 int → S (prod 3)
 3    $ S                  ; x = 1 ;      $ < ;       移进 ;
 4    $ S ;                x = 1 ;        ; < id      移进 x
 5    $ S ; x              = 1 ;          id = =      移进 =
 6    $ S ; x =            1 ;            = < num     移进 1
 7    $ S ; x = 1          ;              num > ;     归约 num → E (prod 25)
 8    $ S ; x = E          ;              = > ;       归约 id = E → S (prod 7)
 9    $ S ; S              ;              ; = ;       移进 ;
10    $ S ; S ;            $              ; > $       归约 S ; → P (prod 2)
11    $ S ; P              $              ; > $       归约 S ; P → P (prod 1)
12    $ P                  $                           接受!
```

### 3.7 语法分析树

分析过程中，每次归约创建一个树节点：左部为父节点，右部符号为子节点。

示例（`int x ; x = 1 ;` 的语法树）：

```
├─ P
│  ├─ S
│  │  ├─ int  (line 1)
│  │  ├─ id  (line 1)
│  ├─ ;  (line 1)
│  ├─ P
│  │  ├─ S
│  │  │  ├─ id  (line 2)
│  │  │  ├─ =  (line 2)
│  │  │  ├─ E
│  │  │  │  ├─ num  (line 2)
│  │  │  ├─ ;  (line 2)
```

## 五、不支持的语法结构及原因

以下语法结构**词法分析器能正确识别 Token**，但**简单优先文法无法处理**。

### 5.1 不支持结构一览

| 结构 | 示例 | 限制原因 |
|------|------|---------|
| 数组声明 | `int a[10]` | `id = [` 与 `E [ E ]` 的 `id > [` 冲突 |
| 成员赋值 | `p.x = 10` | `=` 作为语句中间操作符时，与 `;` 无优先关系 |
| 数组元素赋值 | `a[0] = 5` | 同上 |
| 函数体多语句 | `int main() { s1 ; s2 ; }` | `P → S ; P` 归约时机问题 |

### 5.2 根本原因分析

**原因 1：优先关系表是全局的**

同一个符号对 `(id, [)` 在不同上下文需要不同的关系：
- 数组声明 `int a[10]` 需要 `id = [`
- 数组下标 `a[i]` 需要 `id > [`（先归约 `id` 为 `E`）

优先关系表无法为同一符号对存储两个不同的关系。

**原因 2：只看相邻符号对**

产生式 `S → id = E` 中，`=` 和 `;` 被非终结符 `E` 隔开。算法建立的是 `= < FIRSTVT(E)` 和 `LASTVT(E) > ;`，但 `= < ;` 从未建立。当 `=` 是栈顶终结符且 `;` 是下一个输入时，算法找不到关系。

**原因 3：缺少上下文信息**

`P → S ; P` 归约 `S ;` 为 `P` 时，算法不知道当前是在顶层还是在 `{ }` 块内。在块内，`S ;` 应保留作为 `P → S ; P` 的一部分，但算法过早归约。

### 5.3 解决方向

| 方法 | 能力 | 复杂度 |
|------|------|--------|
| 简单优先文法（本项目） | 基础语法结构 | 低 |
| 算符优先文法 | 运算符优先级处理更好 | 中 |
| LR(1) 分析 | 完整支持所有 CFG | 高 |

## 八、阶段三：语义分析与代码生成

### 8.1 方法

采用**语法制导翻译**（Syntax-Directed Translation）：在语法分析的归约步骤中执行语义动作，生成中间代码，再翻译为目标代码。

### 8.2 中间代码：四元式

四元式格式：`(op, arg1, arg2, result)`

| op | 含义 | 示例 |
|----|------|------|
| `=` | 赋值 | `(=, 10, _, x)` — x = 10 |
| `+,-,*,/` | 算术运算 | `(+, x, 5, t0)` — t0 = x + 5 |
| `<,>,==` | 关系运算 | `(<, x, 10, t1)` — t1 = x < 10 |
| `label` | 标签定义 | `(label, L0, _, _)` — L0: |
| `goto` | 无条件跳转 | `(goto, L0, _, _)` — goto L0 |
| `if_false` | 条件跳转 | `(if_false, t1, L1, _)` — if !t1 goto L1 |
| `param` | 函数参数 | `(param, x, _, _)` — push x |
| `call` | 函数调用 | `(call, write, 1, _)` — call write 1 |
| `return` | 返回 | `(return, 0, _, _)` — return 0 |

### 8.3 目标代码：Jack VM

Jack VM 是基于栈的虚拟机，主要指令：

| 指令 | 含义 | 示例 |
|------|------|------|
| `push constant N` | 压入常量 | `push constant 10` |
| `push/pop local N` | 局部变量读写 | `push local 0` |
| `add/sub/mul/div` | 算术运算 | `add` |
| `eq/lt/gt` | 关系运算 | `lt` |
| `and/or/not` | 逻辑运算 | `not` |
| `label L` | 标签定义 | `label L0` |
| `goto L` | 无条件跳转 | `goto L0` |
| `if-goto L` | 条件跳转 | `if-goto L1` |
| `call f n` | 调用函数 | `call write 1` |
| `return` | 返回 | `return` |

### 8.4 语义动作示例

**赋值语句 `x = 10`：**
```
产生式: S → id = E
语义动作: emit("=", "10", "", "x")
四元式: (=, 10, -, x)
Jack VM: push 10; pop x
```

**算术表达式 `z = x + y * 2`：**
```
归约 y * 2 → E: emit("*", "y", "2", "t0")     // t0 = y * 2
归约 x + E → E: emit("+", "x", "t0", "t1")     // t1 = x + t0
归约 id = E → S: emit("=", "t1", "", "z")       // z = t1

四元式:
  (*, y, 2, t0)
  (+, x, t0, t1)
  (=, t1, -, z)

Jack VM:
  push y; push 2; call Math.multiply 2; pop t0
  push x; push t0; add; pop t1
  push t1; pop z
```

**函数调用 `write(z)`：**
```
产生式: S → id ( E )
语义动作: emit("param", "z"); emit("call", "write", "1")
四元式: (param, z, -, -); (call, write, 1, -)
Jack VM: push z; call write 1
```

### 8.6 已知限制

由于简单优先文法的归约时机限制，**while 循环和 if/else 语句**的控制流代码顺序存在以下问题：

- 条件表达式和循环体/分支体的代码在归约时已生成，导致 `label` 和跳转指令的位置不够精确
- 四元式逻辑正确，但生成的 Jack VM 指令中，条件计算代码出现在循环标签之前
- 这是语法制导翻译与简单优先文法结合的结构性限制，不影响赋值、表达式、函数调用等核心功能的正确性
