# 微型程序语言 (Micro-C) 规范与词法分析设计文档

## 1. 源语言概述

本编译器设计的源语言名为 **Micro-C**，它是 C 语言的一个结构化精简子集。该语言的设计旨在剥离复杂的语法糖，保留结构化编程的核心要素，同时满足将其编译至目标机（Jack 虚拟机，基于栈的指令集）的需求。Micro-C 支持基本的算术运算、逻辑控制流（选择与循环）、基础控制台输入输出，支持自定义复合数据结构 (`struct`)。

## 2. 数据类型

为保证编译器的轻量级与高效性，同时展现处理底层内存寻址的能力，Micro-C 支持以下四种数据类型：

- **`int` (整型)**：基础字长（Word），用于表示整数，支持常规算术与逻辑运算。

- **`char` (字符型)**：处理单个 ASCII 字符（底层以整数形态存储）。

- **`double` (双精度浮点型)**：用于表示浮点数，支持常规算术运算。

- **`struct` (结构体)**：支持用户自定义的纯数据复合类型。编译器将负责计算成员偏移量（Offset），并在目标虚拟机的堆（Heap）内存或局部栈（Stack）上进行相应的基址寻址。


## 3. 词法规范 (Lexical Rules)

词法分析器的任务是从左至右扫描源程序字符流，跳过空白符与注释，将其转换为合法的单词符号（Token）序列。词法集定义如下：

### 3.1 关键字 (Keywords)

语言保留的特殊字符串，严格区分大小写。

- **数据与类型**：`int`, `char`, `double`, `struct`

- **控制流**：`if`, `else` , `while`, `return`

- **系统调用**：`read`, `write`

- **程序入口**：`main`


### 3.2 标识符 (Identifiers)

用于命名变量和结构体类型。

- **构成规则**：必须以英文字母或下划线（`_`）开头，后续跟零个或多个字母、数字或下划线。

- **正则表达式**：`[a-zA-Z_][a-zA-Z0-9_]*`


### 3.3 常数字面量 (Literals)

- **整型常数 (Number)**：连续数字（如 `42`, `1024`）。

- **字符常数 (Char Literal)**：单引号包裹的字符或转义字符（如 `'A'`, `'\n'`）。


### 3.4 运算符 (Operators)

- **算术运算符**：`+`, `-`, `*`, `/`

- **关系运算符**：`>`, `<`, `==`

- **赋值运算符**：`=`

- **成员访问运算符**：`.` (点号，专用于访问结构体成员)


### 3.5 界符 (Delimiters)

- `(`, `)`：表达式分组或函数参数。

- `{`, `}`：定义语句块，控制流严格要求大括号。

- `;`：语句结束标志。
- `[` `]`：数据成员访问。


### 3.6 注释与空白符 (Comments & Whitespaces)

- **单行注释**：`//` 开头。

- **多行注释**：`/*` 开头，`*/` 结尾。

- 扫描时跳过所有空白符（Space, Tab, Newline），但需保留行号和列号信息用于容错追踪。


## 4. 语法结构与综合示例 (Syntax Overview)

Micro-C 程序为语句序列，语句之间以 `;` 分隔。

### 4.1 文法产生式（简单优先文法）

```
P → S ; P | S ;                     (程序 = 语句序列)
S → int id | char id | double id    (变量声明)
  | struct id { P }                 (结构体定义)
  | id = E                          (赋值)
  | id ( E ) | id ( id ) | id ( )   (函数调用: read/write 等)
  | while ( E ) { P }               (循环)
  | if ( E ) { P } else { P }       (选择)
  | return E                        (返回)
E → E + E | E - E | E * E | E / E  (算术)
  | E < E | E > E | E == E          (关系)
  | E . id                          (成员访问)
  | E [ E ]                         (数组下标)
  | ( E ) | id | num | char         (基本元素)
```

### 4.2 支持的语法结构

| 结构 | 示例 |
|------|------|
| 变量声明 | `int x ;` `char c ;` `double d ;` |
| 结构体定义 | `struct Point { int x ; } ;` |
| 结构体变量 | `struct Point p ;` |
| 赋值 | `x = 10 ;` `c = 'A' ;` |
| 算术表达式 | `z = x + y * 2 ;` |
| 关系表达式 | `z = x < y ;` `z = x > b ;` |
| 成员访问 | `z = p . x ;` |
| 函数调用（无参） | `bar() ;` |
| 函数调用（单参） | `read(x) ;` `write(z) ;` `foo(x) ;` |
| 函数调用（双参） | `z = get(a, i) ;` |
| 函数调用（三参） | `set(p, x, 10) ;` |
| 循环 | `while(x < 10) { x = x + 1 ; } ;` |
| 选择 | `if(x > 5) { y = 1 ; } else { y = 2 ; } ;` |
| 返回 | `return 0 ;` |

### 4.4 简单优先文法的限制

以下语法结构在词法层面可正确识别 Token，但简单优先文法无法处理：

| 结构 | 示例 | 限制原因 |
|------|------|---------|
| 成员赋值 | `p.x = 10` | `id . id` 被过早归约为 `E` |
| 数组赋值 | `a[0] = 5` | `id [ E ]` 被过早归约为 `E` |
| 数组声明 | `int a[10]` | `id = [` 与 `E [ E ]` 的 `id > [` 冲突 |
| 函数体多语句 | `int main() { ... }` | `P → S ; P` 在块内的归约时机问题 |

**替代方案：** 上述限制可通过函数调用形式实现：
- 成员赋值 → `set(p, x, 10)` （三参函数）
- 数组赋值 → `set(a, 0, 5)` （三参函数）
- 数组访问 → `z = get(a, i)` （双参函数表达式）

**已实现的功能：**
- ✅ 变量声明：`int x ;` `char c ;` `double d ;`
- ✅ 结构体定义：`struct Point { int x ; } ;`
- ✅ 结构体变量：`struct Point p ;`
- ✅ 赋值：`x = 10 ;` `c = 'A' ;`
- ✅ 算术表达式：`z = x + y * 2 ;`（运算符优先级正确）
- ✅ 关系表达式：`z = x < y ;` `z = x > b ;`
- ✅ 成员访问：`z = p . x ;`
- ✅ 函数调用（无参）：`bar() ;`
- ✅ 函数调用（单参）：`read(x) ;` `write(z) ;` `foo(x) ;`
- ✅ 函数调用（双参）：`z = get(a, i) ;`
- ✅ 函数调用（三参）：`set(p, x, 10) ;`
- ✅ 循环：`while(x < 10) { x = x + 1 ; } ;`
- ✅ 选择：`if(x > 5) { x = 1 ; } else { x = 2 ; } ;`
- ✅ 返回：`return 0 ;`

## 5. 词法分析器 (Lexer) 核心设计与数据结构

本阶段基于 C++ 开发，采用手工编写状态机模式，输出二元式的 Token 流。

### 5.1 Token 结构与类别映射

C++

```
enum class TokenType {
    KEYWORD,      // int, char, struct, if, while...
    IDENTIFIER,   // p, n, Point, label...
    NUMBER,       // 10, 0...
    CHAR_LITERAL, // 'A', 'S'...
    OPERATOR,     // +, -, >, =, .
    DELIMITER,    // {, }, (, ), ;
    UNKNOWN,      // 非法字符
    END_OF_FILE
};

struct Token {
    TokenType type;
    std::string value;
    int line;     // 用于精准定位
    int column;
};
```

### 5.2 容错机制 (Error Handling)

分析器内部集成 `ErrorHandler` 类。当状态机捕获到非法字符或未闭合的字符常数/注释时，系统**不会崩溃或退出**，而是记录异常发生的坐标（line, column）与详情至错误队列，随后重置状态机，丢弃错误字符并继续向后扫描，实现高强度的编译鲁棒性。

### 5.3 符号表管理架构 (Symbol Table Prep)

为支持后续 `struct` 的解析，词法分析阶段初始化的符号表采用分层架构准备：

- 词法层仅负责收集 `IDENTIFIER` 的字面量。

- 预留接口，以便在语法/语义分析阶段建立**类型表 (Type Table)**（记录 `Point` 的总体大小及成员 `x`, `y` 的偏移量）与**变量表 (Variable Table)**（记录 `p` 的类型为 `Point`）。
